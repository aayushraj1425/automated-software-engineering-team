Inside each box

The five-box picture, one box at a time. Each box breaks into 3–4 parts, never more. Under each diagram: which file to open first.

🧑 1. The browser — three screens

flowchart LR
    subgraph SCREENS["What you can open"]
        S1["/chat
talk to one AI model"]
        S2["/runs
start an agent run,
see past runs"]
        S3["/runs/{id}
watch one run:
task board + timeline"]
    end
    S2 -->|"click a run"| S3

Open first: apps/web/src/components/runs/run-detail-panel.tsx — the watch screen, ~100 lines, shows the whole polling idea.

🌐 2. The website — three layers

🤖 5. The AI — one connection, three price tiers

flowchart LR
    RT["llm/router.py"]
    RT -->|"planner — thinking jobs
(plans, reviews)"| M1["gemini-2.5-pro"]
    RT -->|"coder — writing code
(and chat today)"| M2["gemini-2.5-flash"]
    RT -->|"cheap — small jobs
(summaries, routing)"| M3["gemini-2.5-flash-lite"]
    RT -->|"LLM_FAKE=1 — tests,
no key, no cost"| M4["canned replies"]

The tiers are just three lines in .env (MODEL_PLANNER, MODEL_CODER, MODEL_CHEAP). Swap any of them to Claude or GPT and nothing else changes — that's the point of the router.
