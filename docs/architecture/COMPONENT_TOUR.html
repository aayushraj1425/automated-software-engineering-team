Component tour — the whole platform, one figure at a time

High level design first, then the low-level implementation of every component: frontend, backend, database, Docker, GitHub Actions. Companion page: Inside each box.

Part 1 · High Level Design

1.1 The whole system in one picture

The browser never talks to the AI engine directly. The website owns identity and forwards every request with a signed pass; the engine owns everything intelligent; Postgres remembers everything.

1.3 One rule that ties it together: everything runs offline

flowchart LR
    F["LLM_FAKE=1"] --> F1["canned model replies"]
    F --> F2["deterministic fake embeddings
(same text → same vector)"]
    F --> F3["local fixture repository
fixtures/demo-service"]
    F1 & F2 & F3 --> G["CI and your laptop prove the whole
    pipeline with zero API keys and zero cost"]

Real keys only change what the models say, never how the system works.

2.2 Frontend — the relay layer (BFF)

Every src/app/api/* route does the same three steps. The browser never holds an engine credential.

sequenceDiagram
    participant B as Browser
    participant W as Relay route (app/api/chat)
    participant E as Engine (/v1/chat)
    B->>W: fetch with session cookie
    W->>W: 1. validate the session (better-auth)
    W->>W: 2. sign a 60s service JWT (HS256)
    W->>E: 3. forward with Authorization: Bearer
    E-->>W: SSE stream (tokens, citations, done)
    W-->>B: same stream, piped through

Open first: apps/web/src/app/api/chat/route.ts — the whole pattern in ~45 lines. The shared secret is ENGINE_SERVICE_SECRET.

2.4 Backend — the life of an agent run

stateDiagram-v2
    [*] --> queued: POST /v1/runs
    queued --> planning: clone workspace,
PM explores + plans
    planning --> awaiting_approval: plan saved
    awaiting_approval --> executing: you approve
    awaiting_approval --> [*]: you reject
(workspace removed)
    executing --> reviewing: engineers code + commit
(budget checked before each task)
    reviewing --> executing: request changes
(one revision round)
    reviewing --> completed: approve → push branch,
open pull request
    executing --> failed: budget exhausted /
task failed 3×

Open first: apps/engine/src/engine/agents/runner.py. Every tool call lands in the timeline as a tool.called event — arguments summarized, file contents never stored.
